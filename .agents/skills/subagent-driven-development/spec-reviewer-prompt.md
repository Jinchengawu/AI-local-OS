# 规格合规评审提示词模板

派遣规格合规评审子代理时使用此模板。

**目的：** 验证实现者构建了所请求的内容（不多也不少）

```
Task tool (general-purpose):
  description: "评审任务 N 的规格合规性"
  prompt: |
    你正在评审一个实现是否匹配其规格。

    ## 请求的内容

    [任务需求的完整文本]

    ## 实现者声称构建的内容

    [来自实现者的汇报]

    ## 关键：不要信任汇报

    实现者完成得异常迅速。他们的汇报可能不完整、不准确或过于乐观。
    你必须独立验证所有内容。

    **不要：**
    - 相信他们关于实现内容的说法
    - 信任他们关于完整性的声明
    - 接受他们对需求的解读

    **要做的：**
    - 阅读他们实际编写的代码
    - 将实际实现与需求逐条对比
    - 检查他们声称已实现但实际缺失的部分
    - 查找他们未提及的额外功能

    ## 你的工作

    阅读实现代码并验证：

    **缺失的需求：**
    - 他们是否实现了所有请求的内容？
    - 是否有跳过或遗漏的需求？
    - 是否声称某些东西可用但实际未实现？

    **多余/不需要的工作：**
    - 是否构建了未请求的内容？
    - 是否过度设计或添加了不必要的功能？
    - 是否添加了规格中没有的"锦上添花"？

    **理解偏差：**
    - 是否以与预期不同的方式解读了需求？
    - 是否解决了错误的问题？
    - 是否实现了正确的功能但方式不对？

    **通过阅读代码来验证，而非信任汇报。**

    汇报：
    - ✅ 规格合规（如果代码审查后所有内容匹配）
    - ❌ 发现问题：[具体列出缺失或多余的内容，附带 文件:行号 引用]
```
