# 完成开发分支

## 概述

通过提供清晰的选项并处理所选工作流，引导完成开发工作。

**核心原则：** 验证测试 → 展示选项 → 执行选择 → 清理。

**启动时宣告：** "我正在使用 finishing-a-development-branch 技能来完成本次工作。"

## 流程

### 步骤 1：验证测试

**在展示选项前，先验证测试通过：**

```bash
# 运行项目的测试套件
npm test / cargo test / pytest / go test ./...
```

**如果测试失败：**

```
测试失败（<N> 个失败）。完成前必须修复：

[展示失败详情]

在测试通过前无法继续合并/PR。
```

停止。不要进入步骤 2。

**如果测试通过：** 继续步骤 2。

### 步骤 2：确定基础分支

```bash
# 尝试常见的基础分支
git merge-base HEAD main 2>/dev/null || git merge-base HEAD master 2>/dev/null
```

或询问："这个分支是从 main 分出来的——对吗？"

### 步骤 3：展示选项

展示以下 4 个选项：

```
实现已完成。你想怎么处理？

1. 本地合并回 <基础分支>
2. 推送并创建 Pull Request
3. 保持分支现状（稍后自行处理）
4. 丢弃本次工作

选哪个？
```

**不要添加解释** —— 保持选项简洁。

### 步骤 4：执行选择

#### 选项 1：本地合并

```bash
# 切换到基础分支
git checkout <基础分支>

# 拉取最新
git pull

# 合并功能分支
git merge <功能分支>

# 在合并结果上验证测试
<测试命令>

# 如果测试通过
git branch -d <功能分支>
```

然后：清理 worktree（步骤 5）

#### 选项 2：推送并创建 PR

```bash
# 推送分支
git push -u origin <功能分支>

# 创建 PR
gh pr create --title "<标题>" --body "$(cat <<'EOF'
## Summary
<2-3 条变更摘要>

## Test Plan
- [ ] <验证步骤>
EOF
)"
```

然后：清理 worktree（步骤 5）

#### 选项 3：保持现状

报告："保持分支 <名称>。Worktree 保留在 <路径>。"

**不要清理 worktree。**

#### 选项 4：丢弃

**先确认：**

```
这将永久删除：
- 分支 <名称>
- 所有提交：<提交列表>
- Worktree：<路径>

输入 'discard' 确认。
```

等待精确确认。

确认后：

```bash
git checkout <基础分支>
git branch -D <功能分支>
```

然后：清理 worktree（步骤 5）

### 步骤 5：清理 Worktree

**选项 1、2、4：**

检查是否在 worktree 中：

```bash
git worktree list | grep $(git branch --show-current)
```

如果是：

```bash
git worktree remove <worktree路径>
```

**选项 3：** 保留 worktree。

## 快速参考

| 选项        | 合并 | 推送 | 保留 Worktree | 清理分支  |
| ----------- | ---- | ---- | ------------- | --------- |
| 1. 本地合并 | ✓    | -    | -             | ✓         |
| 2. 创建 PR  | -    | ✓    | ✓             | -         |
| 3. 保持现状 | -    | -    | ✓             | -         |
| 4. 丢弃     | -    | -    | -             | ✓（强制） |

## 常见错误

**跳过测试验证**

- **问题：** 合并了有问题的代码，创建了失败的 PR
- **修复：** 在提供选项前始终验证测试

**开放式提问**

- **问题：** "接下来该怎么办？" → 含糊不清
- **修复：** 展示精确的 4 个结构化选项

**自动清理 worktree**

- **问题：** 在可能还需要 worktree 时就删除了（选项 2、3）
- **修复：** 仅在选项 1 和 4 时清理

**丢弃时不确认**

- **问题：** 意外删除工作成果
- **修复：** 要求输入 "discard" 确认

## 红线警告

**绝不：**

- 在测试失败时继续
- 合并后不验证测试结果
- 删除工作成果前不确认
- 未经明确请求就 force-push

**始终：**

- 在提供选项前验证测试
- 展示精确的 4 个选项
- 选项 4 需要输入确认
- 仅在选项 1 和 4 时清理 worktree
