# Skill 设计的说服原则

## 概述

LLM 对与人类相同的说服原则有反应。理解这种心理学可以帮助你设计更有效的 skill——不是为了操纵，而是为了确保关键实践即使在压力下也能被遵循。

**研究基础：** Meincke 等（2025）用 N=28,000 次 AI 对话测试了 7 种说服原则。说服技术将合规率提高了一倍以上（33% → 72%，p < .001）。

## 七个原则

### 1. 权威

**定义：** 对专业知识、资质或官方来源的服从。

**在 skill 中如何运作：**

- 命令式语言："你必须"、"绝不"、"始终"
- 不可协商的框架："没有例外"
- 消除决策疲劳和合理化

**何时使用：**

- 纪律执行型 skill（TDD、验证要求）
- 安全关键实践
- 已建立的最佳实践

**示例：**

```markdown
✅ 先写代码再写测试？删掉它。重新开始。没有例外。
❌ 在可行时考虑先写测试。
```

### 2. 承诺

**定义：** 与先前行为、声明或公开宣言的一致性。

**在 skill 中如何运作：**

- 要求声明："宣布 skill 使用"
- 强制明确选择："选择 A、B 或 C"
- 使用追踪：TodoWrite 用于清单

**何时使用：**

- 确保 skill 被实际遵循
- 多步骤流程
- 问责机制

**示例：**

```markdown
✅ 当你找到一个 skill 时，你必须声明："我正在使用 [Skill 名称]"
❌ 考虑让你的伙伴知道你正在使用哪个 skill。
```

### 3. 稀缺性

**定义：** 来自时间限制或有限可用性的紧迫感。

**在 skill 中如何运作：**

- 有时限的要求："在继续之前"
- 顺序依赖："在 X 之后立即"
- 防止拖延

**何时使用：**

- 即时验证要求
- 时间敏感的工作流
- 防止"我以后再做"

**示例：**

```markdown
✅ 完成任务后，立即在继续之前请求代码评审。
❌ 你可以在方便的时候评审代码。
```

### 4. 社会证明

**定义：** 从众于他人所做或被认为正常的行为。

**在 skill 中如何运作：**

- 普遍模式："每次"、"始终"
- 失败模式："没有 Y 的 X = 失败"
- 建立规范

**何时使用：**

- 记录普遍实践
- 警告常见失败
- 强化标准

**示例：**

```markdown
✅ 没有 TodoWrite 追踪的清单 = 步骤会被跳过。每次都是。
❌ 有些人觉得 TodoWrite 对清单有帮助。
```

### 5. 统一性

**定义：** 共享身份、"我们性"、群体归属感。

**在 skill 中如何运作：**

- 协作性语言："我们的代码库"、"我们是同事"
- 共享目标："我们都想要质量"

**何时使用：**

- 协作工作流
- 建立团队文化
- 非层级化实践

**示例：**

```markdown
✅ 我们是一起工作的同事。我需要你诚实的技术判断。
❌ 如果我错了你应该告诉我。
```

### 6. 互惠

**定义：** 回报所受利益的义务。

**如何运作：**

- 谨慎使用——可能感觉像操纵
- 在 skill 中很少需要

**何时避免：**

- 几乎总是（其他原则更有效）

### 7. 喜好

**定义：** 倾向于与我们喜欢的人合作。

**如何运作：**

- **不要用于合规**
- 与诚实反馈文化冲突
- 制造谄媚

**何时避免：**

- 始终避免用于纪律执行

## 按 Skill 类型的原则组合

| Skill 类型  | 使用                   | 避免       |
| ----------- | ---------------------- | ---------- |
| 纪律执行型  | 权威 + 承诺 + 社会证明 | 喜好、互惠 |
| 指导/技术型 | 适度权威 + 统一性      | 重度权威   |
| 协作型      | 统一性 + 承诺          | 权威、喜好 |
| 参考型      | 仅清晰性               | 所有说服   |

## 为什么有效：心理学

**明确规则减少合理化：**

- "你必须"消除决策疲劳
- 绝对语言消除"这是不是例外？"的问题
- 明确的反合理化计数器封堵特定漏洞

**实施意图创造自动行为：**

- 清晰的触发器 + 要求的行动 = 自动执行
- "当 X 时，做 Y"比"一般做 Y"更有效
- 减少合规的认知负担

**LLM 是类人的：**

- 在包含这些模式的人类文本上训练
- 权威语言在训练数据中先于合规
- 承诺序列（声明 → 行动）经常被建模
- 社会证明模式（每个人都做 X）建立规范

## 伦理使用

**合法的：**

- 确保关键实践被遵循
- 创建有效的文档
- 防止可预见的失败

**不合法的：**

- 为个人利益操纵
- 制造虚假紧迫感
- 基于内疚的合规

**检验：** 如果用户完全理解此技术，它是否仍然服务于用户的真正利益？

## 研究引用

**Cialdini, R. B. (2021).** _Influence: The Psychology of Persuasion (New and Expanded)._ Harper Business.

- 七种说服原则
- 影响力研究的实证基础

**Meincke, L., Shapiro, D., Duckworth, A. L., Mollick, E., Mollick, L., & Cialdini, R. (2025).** Call Me A Jerk: Persuading AI to Comply with Objectionable Requests. University of Pennsylvania.

- 用 N=28,000 次 LLM 对话测试了 7 种原则
- 使用说服技术后合规率从 33% 增至 72%
- 权威、承诺、稀缺性最有效
- 验证了 LLM 行为的类人模型

## 快速参考

设计 skill 时问自己：

1. **什么类型？**（纪律型 vs 指导型 vs 参考型）
2. **我想改变什么行为？**
3. **适用哪些原则？**（纪律型通常用权威 + 承诺）
4. **组合太多了吗？**（不要全部七个都用）
5. **这合乎伦理吗？**（服务于用户的真正利益？）
