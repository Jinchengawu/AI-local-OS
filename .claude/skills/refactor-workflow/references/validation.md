# 技术验证清单

聚焦 lint/build/test 等技术层面的验证。业务逻辑校验参见 `refactor-check` skill。

## 每步检查（必须）

完成每个重构步骤后：

- [ ] 变更文件 eslint 通过（**只检查变更文件**）
- [ ] 无 TypeScript 错误
- [ ] 代码可编译

### 只检查变更文件

**重要：不要直接运行 `pnpm lint`**，它会格式化整个代码库。

```bash
# 获取变更的 ts/tsx 文件
CHANGED_FILES=$(git diff --name-only --diff-filter=AM | grep -E '\.(ts|tsx)$' | tr '\n' ' ')

# 只对变更文件运行 eslint
npx eslint $CHANGED_FILES --fix
```

## 阶段检查

### ANALYZE 阶段完成时

**文件模式**：

- [ ] 已识别所有代码异味
- [ ] 已列出所有依赖文件
- [ ] 已评估风险等级

**模块模式**：

- [ ] 已扫描目录结构
- [ ] 已分析依赖关系（内部/外部/循环）
- [ ] 已对每个文件进行异味检测
- [ ] 已识别模块级问题（缺少分层、导出混乱等）

### PLAN 阶段完成时

**文件模式**：

- [ ] 已分解为小步骤（每步 <50 行）
- [ ] 每步有明确的验证点
- [ ] **已获得用户确认**

**模块模式**：

- [ ] 已按 Phase 分解（基础设施 → 逐文件 → 清理）
- [ ] 文件重构顺序按依赖关系排列
- [ ] **已获得用户确认**

### EXECUTE 阶段完成时

- [ ] 按计划完成所有步骤
- [ ] 每步后已对变更文件运行 eslint
- [ ] 代码始终保持可编译

### VERIFY 阶段完成时

- [ ] 变更文件 eslint 通过
- [ ] `pnpm build` 通过（如有）
- [ ] `pnpm test` 通过（如有）

## 回滚检查

如果重构出问题：

```bash
# 查看最近提交
git log --oneline -5

# 回滚到上一步
git checkout -- <文件>

# 或回滚整个重构
git reset --hard HEAD~1
```

## 质量门禁

**可以继续**：

- 变更文件 eslint 通过
- 无 TypeScript 错误
- 页面可正常渲染

**必须停止**：

- eslint 失败
- TypeScript 报错
- 页面白屏/崩溃
- 测试失败
