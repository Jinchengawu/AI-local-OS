# 代码质量评审提示词模板

派遣代码质量评审子代理时使用此模板。

**目的：** 验证实现构建得是否良好（简洁、有测试、可维护）

**仅在规格合规评审通过后才派遣。**

```
Task tool (code-reviewer):
  使用 requesting-code-review/code-reviewer.md 中的模板

  WHAT_WAS_IMPLEMENTED: [来自实现者的汇报]
  PLAN_OR_REQUIREMENTS: [计划文件] 中的任务 N
  BASE_SHA: [任务开始前的提交]
  HEAD_SHA: [当前提交]
  DESCRIPTION: [任务概述]
```

**除了标准代码质量关注点外，评审者还应检查：**

- 每个文件是否有一个明确的职责和定义良好的接口？
- 单元是否被分解到可以独立理解和测试？
- 实现是否遵循了计划中的文件结构？
- 此次实现是否创建了已经很大的新文件，或显著增大了现有文件？（不要标记已有的文件大小——聚焦于此次变更贡献了什么。）

**代码评审返回：** 优点、问题（严重/重要/次要）、评估
