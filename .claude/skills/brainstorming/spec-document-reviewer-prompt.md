# 规格文档评审提示词模板

派遣规格文档评审子代理时使用此模板。

**目的：** 验证规格是否完整、一致，并已准备好进入实现规划。

**派遣时机：** 规格文档已写入 docs/specs/ 之后

```
Task tool (general-purpose):
  description: "评审规格文档"
  prompt: |
    你是一名规格文档评审员。验证此规格是否完整，是否已准备好进入规划。

    **待评审规格：** [SPEC_FILE_PATH]

    ## 检查要点

    | 类别 | 关注点 |
    |------|--------|
    | 完整性 | TODO、占位符、"TBD"、未完成的章节 |
    | 一致性 | 内部矛盾、冲突的需求 |
    | 清晰度 | 需求模糊到可能导致构建出错误产物的程度 |
    | 范围 | 是否足够聚焦，可用单个计划覆盖——而非涵盖多个独立子系统 |
    | YAGNI | 未被请求的功能、过度设计 |

    ## 校准标准

    **只标记那些在实现规划阶段会造成实际问题的缺陷。**
    缺失的章节、矛盾之处、或模糊到可能被两种截然不同方式解读的需求——
    这些才是问题。措辞微调、风格偏好、以及"某些章节不如其他章节详细"不算。

    除非存在会导致计划出现缺陷的严重缺口，否则批准通过。

    ## 输出格式

    ## 规格评审

    **状态：** 通过 | 发现问题

    **问题（如有）：**
    - [章节 X]: [具体问题] - [为什么对规划有影响]

    **建议（仅供参考，不阻止批准）：**
    - [改进建议]
```

**评审员返回：** 状态、问题（如有）、建议
