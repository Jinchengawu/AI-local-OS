# 计划文档评审提示词模板

派遣计划文档评审子代理时使用此模板。

**目的：** 验证计划是否完整、是否匹配规格、任务分解是否恰当。

**派遣时机：** 完整计划编写完成后。

```
Task tool (general-purpose):
  description: "评审计划文档"
  prompt: |
    你是一名计划文档评审员。验证此计划是否完整，是否已准备好进入实现。

    **待评审计划：** [PLAN_FILE_PATH]
    **参考规格：** [SPEC_FILE_PATH]

    ## 检查要点

    | 类别 | 关注点 |
    |------|--------|
    | 完整性 | TODO、占位符、未完成的任务、缺失的步骤 |
    | 规格对齐 | 计划覆盖了规格需求，无重大范围蔓延 |
    | 任务分解 | 任务有清晰的边界，步骤可操作 |
    | 可构建性 | 工程师能否按此计划执行而不卡住？ |

    ## 校准标准

    **只标记那些在实现阶段会造成实际问题的缺陷。**
    实现者构建了错误的东西或卡住了才是问题。
    措辞微调、风格偏好和"有则更好"的建议不算。

    除非存在严重缺口——规格中缺失的需求、矛盾的步骤、
    占位内容或模糊到无法执行的任务——否则批准通过。

    ## 输出格式

    ## 计划评审

    **状态：** 通过 | 发现问题

    **问题（如有）：**
    - [任务 X，步骤 Y]: [具体问题] - [为什么对实现有影响]

    **建议（仅供参考，不阻止批准）：**
    - [改进建议]
```

**评审员返回：** 状态、问题（如有）、建议
